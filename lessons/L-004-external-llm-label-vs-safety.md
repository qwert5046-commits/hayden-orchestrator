# L-004 — 외부 LLM 라벨 ≠ 시스템 안전 정책 (재분류 규칙)

## 증상
외부 LLM 이 P2(Minor) / P3(Trivial) 로 라벨한 항목이 실제로는 사용자 운영에 큰 영향을 주는 경우가 있음.

## 원인
외부 LLM 의 위험도 라벨은 "코드 품질" 관점이며, "시스템 안전 정책 / 사용자 신뢰도" 관점과 다름.

## 대응 — 안전 재분류 규칙
다음 패턴이면 외부 LLM 라벨이 무엇이든 격상한다:

| 패턴 | 격상 등급 | 사유 |
|---|---|---|
| 안전 안전장치 우회 가능 (예: 기본 dry-run / no-op 모드 우회) | Critical | 의도 정반대 동작 가능 |
| 데이터 파괴 가능 (실수로 prod 데이터 변경 가능) | Critical | 복구 불가 |
| 시크릿 노출 가능 (로그 / 에러 메시지에 키 본문 포함) | Critical | 즉시 회수 필요 |
| 안전 검수 false positive (사용자 신뢰도 즉시 훼손) | Major | 매 산출물에 잘못된 경고 |
| 데이터 union 직렬화로 분류·정확성 손상 | Major | 사용자 의사결정 영향 |
| 모든 산출물에 100% 재현되는 가시적 결함 | Major | 사용자 즉시 인지 |

## 적용
reviewer 가 외부 LLM 결과를 받은 직후 위 표로 자가 점검 → 격상 사유를 `docs/reviews/phase-N.md` 본문에 함께 기록.
